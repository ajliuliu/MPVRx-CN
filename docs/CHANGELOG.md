# 更新日志

以下内容以通俗易懂的方式记录了各版本的实际变化。

## 1.5.0-preview.2 — 预览版

### 📦 MpvLib 更新
- 更新了 mpv 库及其依赖项

### ⚡ 性能与启动优化
- **更快打开视频**：打开视频文件时使用 `Dispatchers.Default` 调用 `playFile()`，避免阻塞 UI 线程，播放器启动更快
- **更轻量的启动同步**：MPV 目录同步不再每次启动时全量复制 `shaders/` 和 `fonts/`，仅预先同步配置文件、脚本和 `script-opts/`。`mpv.conf` 中引用的着色器通过 `syncReferencedShaders()` 按需拉取，字体由字体管理器处理。对于着色器包较大的用户，启动时间明显缩短
- **移除动态速度叠加层**：旧的 `SpeedControlSlider`（全屏圆点轨道滑块）和 `CompactSpeedIndicator` 已移除。长按加速现在显示为简洁的文字标签（如 "2x"），`showDynamicSpeedOverlay` 偏好设置也已移除
- **长按速度吸附预设值**：长按加速手势现在吸附到固定预设值（0.5x → 1x → 1.5x → 2x → 2.5x → 3x → 3.5x → 4x），不再是自由滑动。设置中的滑块也会吸附到这些值
- **加速范围上限调整**：加速速度上限从 6x 调整为 4x（范围 0.5x–4x）

### 📱 平板双窗格布局
- **文件夹双窗格视图**：在平板设备上（最小屏幕宽度 ≥ 600dp），可在左侧查看文件夹列表，右侧查看视频列表。点击文件夹即可在右侧看到其内容。外观设置中新增"双窗格视图"开关
- **设置双窗格视图**：设置界面在平板上也采用双面板布局，左侧为分类列表，右侧显示对应设置页，当前选中的分类有高亮背景
- **双窗格返回导航**：双窗格模式下按返回键取消选中文件夹/设置页，而非关闭整个界面

### 🎥 播放器与字幕
- **反转字幕滑动方向**：手势偏好中新增"反转字幕滑动方向"设置。启用后从左向右滑动快退，从右向左滑动快进
- **截图文件名模板重做**：
  - `%wH`、`%wM`、`%wS`、`%wT` 现在使用视频播放位置（而非系统时钟），截图文件名与实际视频时间戳一致
  - 新增模板占位符：`%F`（不含扩展名的文件名）、`%P`（位置，格式为 `HH:MM:SS.mmm`）、`%p`（位置，格式为 `HH:MM:SS`）
  - `%f` 优先使用实际文件名，回退到媒体标题，命名更可靠
- **韩文谚文字幕修复**：下载的韩文谚文（组合字符）字幕现在经过 NFC Unicode 规范化处理，不再出现损坏的韩文字符
- **字幕搜索键盘修复**：为 PlayerActivity 添加 `android:windowSoftInputMode="adjustResize"`，字幕搜索对话框不再被虚拟键盘遮挡
- **字幕持久性修复**：外部字幕和字幕设置现在能在播放会话之间更可靠地保留。新增 `addSubtitleSuspend()`（挂起版本）以改善协程处理
- **缓存指示器修复**：进度条上的缓冲范围不再重复计算已播放部分，现在正确显示播放头前方的剩余缓冲
- **颜色十六进制修复**：`toColorHexString()` 现在手动提取 ARGB 分量，解决了某些颜色使用 `Int.toHexString()` 产生错误值的问题

### 🛠️ Lua 脚本改进
- **Lua `require()` 支持**：使用 `require()` 的自定义 Lua 脚本现在可以在 `script-modules/` 子目录中找到模块。应用会递归地将 `scripts/` 中的辅助文件夹同步到内部存储，以便 Lua 的 C 级 `fopen()` 能够读取。在设置中禁用脚本时，模块也会被清理

### 🧹 清理
- **移除 libpython 二进制文件**：从所有 4 个架构中删除了约 50 MB 未使用的 `libpython_bin.so` 文件
- **简化 MPV 版本显示**：移除了 CrashActivity 中的 `cleanBundledMpvVersion()` 方法，MPV 版本现在直接显示，无需字符串修补
- **字体文件夹不再自动设置**：更改基础存储根目录时，字体文件夹偏好设置不再被盲目覆盖，仅当指向旧根目录时才清除，防止意外重置

## 1.5.0-preview.1 — 预览版

### 📦 MpvLib 更新
- 更新了 mpv 库及其依赖项

### ⚡ 性能与稳定性
- **启动优化**：延迟冷启动数据库初始化、语法预加载和自动更新检查，显著缩短首帧时间
- **内存泄漏修复**：修复了 5 处内存泄漏，涉及 PlayerActivity（屏幕状态接收器）、PlayerViewModel（LRU 缓存、临时字幕清理）、MediaPlaybackService（位图泄漏）、MainActivity（未使用的作用域）和 NetworkLifecycleObserver（未取消的协程）
- **UI 流畅度**：优化了进度条弹簧动画、预计算跳过分段颜色、PlayerControls 中的不可变副本记忆化、提升了卡片级偏好收集器
- **播放器崩溃修复**：解决了 PlayerActivity 在流初始化期间的崩溃问题
- **流媒体优化**：在 `Dispatchers.IO` 而非 Default 上加载网络流，减少 CPU 池争用

### 🎨 UI/UX 改进
- **动态网格**：添加响应式网格布局，根据屏幕宽度自动调整列数，覆盖 FileSystemBrowser、FolderList、VideoList、Playlist 和 RecentlyPlayed 界面
- **居中控件**：竖屏模式下播放/暂停和导航按钮居中显示
- **触觉反馈**：排序对话框滑块在吸附时提供触觉振动
- **滑块布局**：列数滑块并排放置，更好地利用空间
- **主题刷新**：提升所有主题（浅色/深色）的容器饱和度和表面色调，视觉效果更鲜艳
- **字幕面板重新设计**：在 OnlineSubtitleSearchSheet 搜索行中直接内联 SeriesSelectionControls
- **压缩器返回键修复**：视频压缩器叠加层现在正确处理系统返回键

### 🎥 播放器与流媒体
- **HLS/DASH 流修复**：直接的媒体 URL（.m3u8/.mpd/.mp4/.ts）现在绕过 yt-dlp，使用 mpv 的原生 ffmpeg HLS 解复用器
- **yt-dlp 音频选择**：yt-dlp 面板中新增音轨选择器，优化首选语言自动选择
- **可折叠高级设置**：YtdlpSettingsScreen 现在将高级选项组织在可折叠区域下
- **控制台选项**：MoreSheet 统计行中新增控制台开关，通过 script-message 打开 mpv 调试控制台
- **字幕触摸区域修复**：多行换行文本的动态触摸区域检测；修复了缩放/平移下的触摸区域；降低了最小字幕缩放限制
- **字幕加载修复**：修复了 IntentSubtitleLoadPolicy 和 M3uPlaybackPolicy 的字幕加载和播放器叠加层问题
- **负亮度支持**：垂直滑块中新增负亮度范围支持
- **流媒体叠加层修复**：修复了流媒体播放叠加层和缩略图渲染

### 🔍 Anime Skip 提供商
- 新增 **Anime Skip** 提供商（api.anime-skip.com GraphQL），用于片头/片尾检测，通过 MAL ID 搜索，配对连续时间戳
- 移除了无效的 db.videasy.net TMDB 镜像
- 清理了 WyzieSearchRepository：移除回退镜像逻辑和无效的 VideasyTmdbTrendingResponse

### 📁 媒体搜索引擎
- **新 MediaSearchEngine**：专用搜索引擎，用于搜索文件/文件夹并优化索引
- **FolderListScreen 集成**：使用新引擎和 VideoFolder 引用跨文件夹和视频搜索
- **多项重构**：清理搜索逻辑，改进可读性，添加 Kotlin 智能转换说明

### 🐛 Bug 修复
- 修复了 mpv 控制台 ytdl_hook 警告（移除无效的 'all_subtitles' 选项）
- 修复了 video-aspect-override 弃用警告
- 解决了流播放期间的播放器崩溃问题
- 修复了字幕滑动手势触摸区域
- 修复了动态网格渲染的边缘情况
